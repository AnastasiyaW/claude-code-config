# 🔴 КАЧЕСТВО > СКОРОСТЬ. БЕЗ МАНКИ-ПАТЧЕЙ.

Директива пользователя (2026-06-07):
> «мы НЕ делаем манки-патчи без КРАЙНЕЙ нужды. стараемся всё делать качественно и в лучших
> традициях разработки.»

## Правило
- **Манки-патч (monkey-patch) запрещён** без КРАЙНЕЙ нужды. Между костылём и чистым
  переписыванием — **всегда переписываем** (чисто, с пониманием причины), потом **верифицируем**.
- Между быстрым хаком и правильной архитектурой — **правильная архитектура**.
- Сложность оправдана, только когда даёт **больше стабильности**, а не просто «работает».
- После любого переписывания — **обязательная проверка**: тест / запуск / чтение diff (durable
  artifact, не слова).

## Что считается «крайней нуждой» (тогда можно патч, с пометкой)
- Прод горит, нужен немедленный hot-fix → патч + сразу заведённый follow-up на чистое решение.
- Внешняя библиотека, которую нельзя править → точечный shim с комментарием-обоснованием.
В остальных случаях — переписываем нормально.

## Override
Только явный OK пользователя на конкретный случай. «Так быстрее» / «и так сойдёт» — не основания.
